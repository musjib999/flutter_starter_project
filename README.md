# first_app

A Flutter project scaffolded with [Mason](https://docs.brickhub.dev) using the `flutter_starter_project` brick.

## Prerequisites

- [Flutter SDK](https://docs.flutter.dev/get-started/install) installed and on your `PATH`
- Dart SDK (included with Flutter)

## 1. Install the Mason CLI

Activate Mason globally via Dart pub:

```bash
flutter pub global activate mason_cli
```

Ensure the Dart global bin directory is on your `PATH` (add to `~/.zshrc` if needed):

```bash
export PATH="$PATH:$HOME/.pub-cache/bin"
```

Verify the installation:

```bash
mason --version
```

## 2. Initialize Mason in a Flutter project

From the root of your Flutter project (where `pubspec.yaml` lives):

```bash
cd /path/to/your/flutter/project
mason init
```

This creates a `mason.yaml` file in the project root.

## 3. Register the `flutter_starter_project` brick

Open `mason.yaml` and add the brick under `bricks:`.

### Option A: Local path (development)

Use this when the brick lives on your machine:

```yaml
bricks:
  flutter_starter_project:
    path: /Users/mac/Documents/dev/template/flutter_starter_project
```

> **Tip:** Use an absolute path, or a relative path from the project root (e.g. `../template/flutter_starter_project`).

### Option B: Git remote (team / CI)

Use this when the brick is hosted in a repository:

```yaml
bricks:
  flutter_starter_project:
    git:
      url: https://github.com/your-org/flutter_starter_project.git
      path: .
```

## 4. Install the brick

Download and cache all registered bricks:

```bash
mason get
```

Run this again whenever `mason.yaml` changes or the brick template is updated.

## 5. Generate the project scaffold

Run the brick from the project root:

```bash
mason make flutter_starter_project
```

Mason will prompt for variables defined in the brick. The current brick supports:

| Variable   | Type   | Default  | Description              |
|------------|--------|----------|--------------------------|
| `appTitle` | string | `My App` | The title of your app    |

### Non-interactive usage

Pass variables on the command line:

```bash
mason make flutter_starter_project --appTitle "My App"
```

### Handling file conflicts

If files already exist (e.g. `lib/main.dart`), choose a conflict strategy:

```bash
# Always overwrite existing files
mason make flutter_starter_project --on-conflict overwrite

# Always skip existing files
mason make flutter_starter_project --on-conflict skip

# Prompt for each conflict (default)
mason make flutter_starter_project --on-conflict prompt
```

## 6. Post-generation setup

The brick runs a **post-gen hook** that automatically:

- Adds dependencies (`dio`, `get_it`, `google_fonts`, `flutter_svg`, etc.)
- Adds `flutter_gen` configuration to `pubspec.yaml`
- Registers asset paths under `assets/images/`
- Runs `dart run build_runner build -d` to generate asset code

If `build_runner` fails, run it manually:

```bash
dart run build_runner build -d
```

Then fetch dependencies if needed:

```bash
flutter pub get
```

## What gets generated

The brick scaffolds a starter architecture under `lib/`:

```
lib/
├── core/
│   ├── gen/          # Generated by flutter_gen
│   └── utils/        # Locator, string/date/media helpers
├── data/
│   └── source/       # Local & remote data layers
├── features/
│   └── user/         # Example feature model
├── shared/
│   ├── components/   # Buttons, dialogs, input fields
│   └── themes/       # App theme, colors, text styles
└── main.dart
```

Assets are placed in `assets/images/`.

## Quick reference

```bash
# Full setup from scratch
flutter pub global activate mason_cli
cd /path/to/your/flutter/project
mason init
# Edit mason.yaml to register flutter_starter_project
mason get
mason make flutter_starter_project --appTitle "My App" --on-conflict overwrite
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `mason: command not found` | Add `$HOME/.pub-cache/bin` to your `PATH` and restart the terminal |
| `Missing subcommand for "mason make"` | Specify the brick name: `mason make flutter_starter_project` |
| Brick not found after editing `mason.yaml` | Run `mason get` again |
| `build_runner` fails | Run `flutter pub get`, then `dart run build_runner build -d` |
| File conflicts during generation | Use `--on-conflict overwrite` or `skip` |
